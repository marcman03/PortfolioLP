Decentralized Voting System

I envisioned developing a decentralized voting system using blockchain technology, but for this project, I implemented a conventional database to explore what kind of data would need to be stored. This helped me understand the structure required for a future blockchain implementation.

Objectives
  • Create an intuitive interface to manage large sets of data.
  • Develop a script to generate and populate a database with millions of records.
  • Simulate a secure voting system where votes can be reviewed on a "blockchain".
  • Create two user roles: one who can generate votes (administrator) and another who can vote (member).
What I Learned

One of the most valuable lessons I learned in this project was how to efficiently handle large amounts of data. By developing a script that generated millions of records and implementing a system capable of managing these volumes, I gained a deep understanding of the challenges and strategies involved in large-scale data management.

Technologies Used
  • Node.js
  • Express.js
  • Vue.js
  • PostgreSQL
  • JavaScript
  • Python

Checkers AI

In this project, I developed an AI to play checkers using the Minimax algorithm with enhancements like Iterative Deepening Search (IDS) and Zobrist hashing for board state optimization. The project was done in collaboration with a friend, based on an environment provided by my professor, which included a pre-built checkers board and various helper functions to manage the game's logic and state. This foundation allowed us to focus on implementing and optimizing the AI algorithms to make the player more competitive.

Objectives
  • Implement the Minimax algorithm with alpha-beta pruning for decision-making in a checkers game.
  • Create two AI players: one using Iterative Deepening Search (IDS) with a time constraint and another using a fixed depth limit for the Minimax search.
  • Optimize board evaluations using Zobrist hashing to speed up move generation and avoid redundant calculations.
  • Design a robust heuristic based on piece counting, triangular formations, and trap identification to improve AI decision-making.
What I Learned

This project taught me how to efficiently apply the Minimax algorithm in a real-world scenario by integrating optimization techniques such as Zobrist hashing and IDS. I learned how to handle time constraints while maintaining competitive performance and how to balance the trade-offs between search depth and computational speed.

Technologies Used
  • Java
  • Minimax Algorithm
  • Alpha-beta Pruning
  • Iterative Deepening Search (IDS)
  • Zobrist Hashing
